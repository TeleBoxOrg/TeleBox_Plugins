## 插件介绍

PMCaptcha 是一款用于 Telegram UserBot 的私聊人机验证插件，旨在自动过滤机器人、广告账号和陌生骚扰消息。

**核心能力：**

陌生人私聊时，插件会自动将对话静音并归档，根据配置向对方发送验证题。验证通过后自动放行，失败或超时后可执行屏蔽、举报、删除对话等操作。整个流程无需人工干预。

**验证模式支持：**

提供四种验证模式——算术验证（随机四则运算/幂运算）、关键词验证（回复指定词或内置随机常识问答）、图片验证纯数字版、图片验证字母+数字版。图片模式会自动安装所需的 `canvas` 依赖，无需手动操作。

**灵活的动作配置：**

验证失败和通过后的操作均可自由组合，支持屏蔽、举报、删除对话、取消静音、加入白名单等，设置时英文名和中文名均可识别。修改配置后，所有正在进行中的验证消息会即时刷新，对方看到的提示同步更新。

**数据安全：**

用户数据（白名单、验证记录）存放在独立文件，更新插件不会丢失。我方主动发起对话时，对方自动标记为已验证，无需手动操作。

| 功能 | 说明 |
|------|------|
| 🚦 自动拦截 | 陌生人私聊时自动静音 + 归档，触发验证流程 |
| 🧮 多种验证模式 | 算术验证、关键词验证、图片验证（纯数字 / 字母+数字）|
| ⏱ 超时控制 | 可设置验证超时秒数，超时自动执行失败操作 |
| 🔢 次数限制 | 可设置最多尝试次数，耗尽后执行失败操作 |
| 🔄 实时更新 | 修改参数后，正在进行中的验证消息即时刷新 |
| 🛡 失败操作 | 屏蔽 / 删除对话 / 举报 / 静音 / 归档，支持复选 |
| ✅ 通过操作 | 取消静音 / 取消归档 / 加入白名单，支持复选 |
| 📋 白名单 | 手动添加信任用户，永久免验证 |
| 📊 验证记录 | 记录通过 / 失败历史，含失败原因，支持按用户删除 |
| 💬 中文支持 | 失败/通过操作支持中文名称输入和显示 |
| 🖼 图片验证码 | 自动安装 canvas 依赖，无需手动操作 |

## 工作原理

```
陌生人发来私聊
       │
       ▼
  在白名单？ ──是──▶ 直接放行
       │否
       ▼
  已通过验证？──是──▶ 直接放行
       │否
       ▼
  静音 + 归档
       │
  验证功能开启？──否──▶ 静默等待（不发验证题）
       │是
       ▼
  发送验证题（算术/关键词/图片）
       │
  ┌────┴────┐
  │         │
答对       答错 / 超时 / 次数耗尽
  │         │
放行     执行失败操作
  │     （屏蔽/举报/删除等）
  ▼
执行通过操作
（取消静音/加白名单等）
```

**我方主动发起对话时**，对方会被自动标记为已验证，无需对方完成验证。

## 验证模式

### `math` — 算术验证（默认）

随机生成四则运算、混合运算、幂运算等，答案均为正整数，无需额外依赖。

```
🔒 人机验证

请回复以下算式的答案：

7 × 13 + 5 = ?

⏱ 验证时间：30 秒
🔢 剩余次数：3 次
⚠️ 验证失败将会：屏蔽
```

题型包括：加法、减法、乘法、整除除法、两步混合运算（a×b+c）、平方。

---

### `text` — 关键词验证

- **默认关键词**（`我同意`）未修改时：从内置题库随机抽取中文常识问题
- **自定义关键词**后：要求对方回复指定词语，不区分大小写

```
🔒 人机验证

请回复以下关键词以证明你不是机器人：

同意接受隐私政策

⏱ 验证时间：30 秒
🔢 剩余次数：3 次
⚠️ 验证失败将会：仅归档并静音
```

---

### `img_digit` — 图片验证（纯数字）

生成含干扰线、噪点、色块的 5 位数字图片验证码，容错 1 个字符。

首次使用会**自动安装 `canvas` npm 模块**，无需手动操作。

---

### `img_mixed` — 图片验证（字母+数字）

同上，字符集为大写字母+数字（已去除易混淆的 `0/O`、`1/I`），共 5 位。

## 失败 & 通过操作

### 失败操作（`set fail`）

验证失败时，**始终执行静音 + 归档**，以下为可选附加操作，支持复选：

| 英文 | 中文 | 说明 |
|------|------|------|
| `block` | `屏蔽` | 屏蔽用户 |
| `delete` | `删除` | 双方撤回全部对话记录 |
| `report` | `举报` | 举报为垃圾信息 |
| `mute` | `静音` | 永久静音（失败时已默认执行，配置幂等） |
| `archive` | `归档` | 归档（失败时已默认执行，配置幂等） |
| `none` | `无` | 清除所有附加操作，仅保留默认静音+归档 |

### 通过操作（`set pass`）

验证通过后执行，支持复选：

| 英文 | 中文 | 说明 |
|------|------|------|
| `unmute` | `取消静音` | 取消对该用户的静音 |
| `unarchive` | `取消归档` | 将对话从归档移出 |
| `wl` | `白名单` | 加入白名单，后续免验证 |
| `none` | `无` | 清除所有通过操作 |

## 指令参考

> 以下示例以 `.` 为命令前缀，实际前缀取决于框架配置。
> `.pmc` 与 `.pmcaptcha` 完全等价，互为别名。

### 插件开关

```bash
.pmc on          # 启用插件（开始拦截陌生人私聊）
.pmc off         # 禁用插件（停止拦截，验证配置保留不变）
.pmc status      # 查看完整配置与运行状态
```

> ⚠️ `off` 只停用插件本身，不会重置验证（captcha）的开/关状态。
> 下次 `on` 后，验证功能自动恢复到关闭前的状态。

---

### 验证设置

```bash
# 开关验证功能
.pmc captcha on
.pmc captcha off

# 切换验证模式（直接写模式名即可）
.pmc captcha math        # 算术验证（默认）
.pmc captcha text        # 关键词验证
.pmc captcha img_digit   # 图片验证（纯数字）
.pmc captcha img_mixed   # 图片验证（字母+数字）
```

---

### 参数配置

```bash
# 超时与次数
.pmc set time 60          # 设置验证超时为 60 秒（0 = 不限时，默认 30）
.pmc set tries 5          # 设置最大尝试次数为 5 次（0 = 不限次，默认 3）

# 文字模式关键词
.pmc set keyword 我已阅读并同意

# 自定义提示语（留空恢复默认）
.pmc set prompt 请回答：{question}   # math 模式占位符
.pmc set prompt 请回复：{keyword}    # text 模式占位符
.pmc set prompt                      # 留空 = 恢复默认提示语

# 失败操作（英文/中文均可，可复选，空格分隔）
.pmc set fail block report           # 英文
.pmc set fail 屏蔽 举报              # 中文
.pmc set fail none                   # 清除所有附加操作

# 通过操作（英文/中文均可，可复选）
.pmc set pass unmute unarchive       # 英文
.pmc set pass 取消静音 取消归档      # 中文
.pmc set pass wl                     # 通过后加白名单
.pmc set pass none                   # 清除所有通过操作
```

> 修改 `fail`、`time`、`tries` 后，**所有正在进行中的验证消息会即时刷新**，
> 对方看到的提示内容同步更新。

### 白名单管理

```bash
# 快捷命令
.pmc add 123456789          # 通过 ID 加入白名单
.pmc add @username          # 通过用户名加入白名单
.pmc add                    # 回复对方消息时无需提供 ID
.pmc del 123456789          # 从白名单移除

# wl 子命令（完整版）
.pmc wl                     # 查看白名单列表
.pmc wl add 123456789
.pmc wl del 123456789
.pmc wl del all             # 清空整个白名单
.pmc wl pass 123456789      # 手动标记为通过验证并加入白名单
```

---

### 验证记录

```bash
.pmc record                           # 查看通过/失败人数摘要
.pmc record verified                  # 查看验证通过记录列表
.pmc record failed                    # 查看验证失败记录（含失败原因）

.pmc record del verified 123456789    # 删除指定用户的通过记录
.pmc record del verified all          # 清空所有通过记录
.pmc record del failed 123456789
.pmc record del failed all
```

失败记录包含原因标注：`⏰ 超时` 或 `❌ 次数耗尽`。

## 数据存储

插件使用两个独立的 JSON 文件存储数据：

| 文件 | 路径 | 内容 | 更新插件后 |
|------|------|------|-----------|
| `pmcaptcha_config.json` | `assets/pmcaptcha/` | 插件配置（模式、超时、操作等）| 可能被重置 |
| `pmcaptcha_data.json` | `<工作目录>/pmcaptcha_userdata/` | 白名单、验证记录 | **不会丢失** |

用户数据（白名单、记录）存放在工作目录下的独立文件夹，更新插件不会覆盖。

## 注意事项

- **图片验证码**：首次使用 `img_digit` 或 `img_mixed` 模式时自动安装 `canvas` 模块，安装期间该用户会收到"验证服务暂时不可用"提示，安装完成后恢复正常。
- **`pmc off` 行为**：只停用插件本身，不重置验证开关状态；下次 `pmc on` 后验证功能自动恢复到之前的状态。
- **实时刷新范围**：修改 `fail`/`time`/`tries` 后，正在等待验证的用户看到的消息即时更新；但超时定时器的倒计时不会因修改超时值而重置，以发送验证时的配置为准。
- **我方主动发起对话**：对方自动记录为已验证，无需手动添加白名单。
- **kick / ban 操作**：在 FailAction 中有定义，私聊场景下无实际效果，仅对群组有意义。
